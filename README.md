# Xplooria Flutter App

Minimal gamified Flutter mobile app for discovering tourist spots in Mizoram.  
Connects to the **SpotMizoram Next.js backend** via its REST API.

---

## Architecture

```
lib/
├── main.dart                   # App entry point (ProviderScope + Firebase init)
├── firebase_options.dart       # ⚠️ Generated by FlutterFire CLI (see Setup)
│
├── models/                     # Pure data classes (fromJson / toJson)
│   ├── user_model.dart         # UserModel — XP, level, badges, bookmarks
│   ├── spot_model.dart         # SpotModel — location, rating, images
│   ├── gamification_models.dart # LevelInfo, LeaderboardEntry, BadgeModel
│   └── community_models.dart   # CommunityPost, BucketList, Dilemma
│
├── services/                   # API calls (Dio)
│   ├── api_client.dart         # Dio singleton + Firebase ID token interceptor
│   ├── auth_service.dart       # Firebase Auth + /api/auth/me
│   ├── spots_service.dart      # /api/spots, /api/reviews, bookmarks
│   └── community_service.dart  # /api/community/*, /api/leaderboard, /api/contributions
│
├── controllers/                # Riverpod state (Notifier / FutureProvider)
│   ├── auth_controller.dart    # AuthState — login, register, Google, profile update
│   ├── spots_controller.dart   # Paginated spots, featured, search, bookmarks
│   └── community_controller.dart # Posts, dilemmas, leaderboard
│
├── core/
│   ├── theme/app_theme.dart    # Dark-first design system (AppColors + AppTheme)
│   ├── constants/app_constants.dart # API base URL, route strings, categories
│   └── router/app_router.dart  # GoRouter with auth redirect + AppRoutes
│
├── screens/
│   ├── onboarding/             # 3-page onboarding carousel
│   ├── auth/                   # login, register, forgot_password
│   ├── shell/                  # Bottom nav shell (main_shell.dart)
│   ├── home/                   # Featured spots + categories + XP card
│   ├── spots/                  # Paginated list + detail with gallery & reviews
│   ├── search/                 # Search with category filter + shimmer
│   ├── community/              # Posts | Bucket Lists | Dilemmas tabs
│   ├── leaderboard/            # Podium + ranked list
│   ├── profile/                # Stats, badges grid, saved spots, edit profile
│   └── contribute/             # Multi-step new spot submission form
│
└── widgets/
    ├── spot_cards.dart         # SpotCard, FeaturedSpotCard, CompactSpotCard
    └── gamification_widgets.dart # XpProgressBar, LevelBadge, BadgeChip, etc.
```

## Setup

### 1. Create a new Flutter project

```bash
flutter create spot_mizoram --org com.yourcompany
cd spot_mizoram
```

### 2. Replace `lib/` and `pubspec.yaml`

Copy the entire `flutter_app/lib/` folder and `flutter_app/pubspec.yaml`  
into your new project, replacing the defaults.

```bash
cp -r /path/to/spotmizoram_2026/flutter_app/lib  spot_mizoram/
cp    /path/to/spotmizoram_2026/flutter_app/pubspec.yaml  spot_mizoram/
```

### 3. Create asset directories

```bash
mkdir -p spot_mizoram/assets/{images,lottie,icons}
```

### 4. Install dependencies

```bash
cd spot_mizoram
flutter pub get
```

### 5. Configure Firebase

Install the FlutterFire CLI (once):

```bash
dart pub global activate flutterfire_cli
```

Run configure inside your Flutter project:

```bash
flutterfire configure
```

Select your Firebase project and the platforms you want (Android, iOS).  
This overwrites `lib/firebase_options.dart` with real values.

### 6. Configure Google Sign-In (Android)

Add your `google-services.json` (downloaded from Firebase Console) to:

```
android/app/google-services.json
```

For iOS, add `GoogleService-Info.plist` via Xcode.

### 7. Set the API base URL

Edit `lib/core/constants/app_constants.dart`:

```dart
static const String apiBaseUrl = String.fromEnvironment(
  'API_BASE_URL',
  defaultValue: 'http://10.0.2.2:3000',   // Android emulator
  // defaultValue: 'http://localhost:3000', // iOS simulator
  // defaultValue: 'https://your-app.vercel.app', // Production
);
```

Or pass at build time:

```bash
flutter run --dart-define=API_BASE_URL=https://your-app.vercel.app
```

### 8. Run

```bash
flutter run
```

## Gamification System

| Action                 | XP Reward |
| ---------------------- | --------- |
| Daily login            | +2        |
| Write a review         | +10       |
| Review with photo      | +15       |
| Submit a new spot      | +10       |
| Spot approved by admin | +100      |

## Notes

- `firebase_options.dart` is a **placeholder** — run `flutterfire configure` before building.
- The Next.js backend must be running locally (`npm run dev`) or deployed to Vercel.
- For production, set `API_BASE_URL` to your Vercel deployment URL.
- Admin-only features (spot approval, user management) live in the Next.js web app.
